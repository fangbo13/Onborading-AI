## 当前产品定位

KnowPilot V6.0 是 RAG 驱动的知识库应用 Agent。它不再只服务“新员工入职问答”，而是作为组织级知识库入口，承载文档入库、向量检索、对话问答、来源引用、权限管控和审计追踪。

它面向的核心痛点是：专业知识分散、答案口径不统一、项目经验难沉淀、人员流动造成知识断层。KnowPilot 将文档、制度、方法论、历史项目判断和培训资料统一纳入知识库，通过对话式 Agent 输出带来源的答案，让团队成员不再依赖“找人问”和“翻文件夹”来获取关键知识。

因此，KnowPilot 不只是一个单点问答工具，而是可以推广复用的知识应用底座。同一套引擎能够按团队或业务场景复制：入职培训换成 HR 知识库，审计项目换成底稿与项目经验库，准则问答换成法规条文库。只要持续维护知识库，就能在多个团队中低成本复制，适合作为组织级知识管理与专业服务交付效率提升工具推广。

当前能力可以映射为四层：

| 层级 | 模块 | 说明 |
|------|------|------|
| 对话入口 | 聊天界面、会话管理、新手引导 | 用户通过自然语言向知识库 Agent 提问，获得流式答案与来源引用 |
| 知识底座 | 知识库管理、RAG、Embedding、pgvector | 文档上传、索引、召回、重排与引用生成 |
| 组织治理 | 认证登录、个人资料、权限控制、审计日志 | 控制谁能访问、管理和追踪知识资产 |
| 体验基础 | 主题、国际化、响应式、错误处理与可访问性 | 保证跨设备、双语和异常场景下的稳定使用体验 |

适用场景包括入职培训、审计方法论查询、准则问答、项目经验沉淀、风险识别、底稿编制、独立性合规检查与审计意见决策。历史来源文档中的 “EY Onboarding AI” 保留为版本演进记录；V6.0 及后续新文档和对外表达统一使用 KnowPilot。

## V6.0 版本口径

V6.0 的功能文档口径从“单一 onboarding chatbot”升级为“知识库应用 Agent 平台”。因此，历史的聊天、会话、知识库管理、权限、审计、主题、国际化等模块继续保留，但解释方式按以下 V6.0 能力分组：

| V6.0 能力域 | 已有模块承接 | 当前说明 |
|-------------|--------------|----------|
| 知识库 Agent 问答 | 模块 2、模块 3、RAG 基础设施 | 以对话作为知识库入口，支持流式回答、来源引用、会话沉淀与知识追溯 |
| 知识资产管理 | 模块 5、Embedding、pgvector、Celery | 支持文档上传、索引、重建、删除、状态追踪和 Web 采集入库 |
| 多团队知识空间 | 认证、个人资料、权限控制、RBAC 规划 | 面向不同业务线和团队复用同一套知识引擎，按角色治理访问边界 |
| 管理与合规审计 | Audit、系统监控、权限管控 | 保留关键操作日志、角色治理与合规追踪能力 |
| 企业级体验 | 主题、i18n、响应式、错误处理 | 支撑中英双语、跨设备使用、可访问性与异常场景恢复 |
